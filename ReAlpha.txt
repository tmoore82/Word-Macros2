'Works well, but also creates a lot of clutter in 
'really problematic documents. Not sure which is 
'worse: nasty conversion, or nasty conversion with 
'highlighting and bookmarks of all the problem areas.

'ReAlpha works on the same principle as DeHyphenate,
'but it has to be much more discerning.
'PDF-to-Word conversions often contain a high number
'of mistaken letters, such as "rn" for "m," or
'"nn" for "rm." Much of this depends on the quality
'of the original PDF.

'ReAlpha takes some of the most commmon mistaken letter combos
'and does its best to recognize where they are wrong
'so it can fix them. So it might look for "nn," select the word
'that contains it, and check the dictionary. If the word is
'"sunny," the macro should leave it alone. If the word is
'"tenn," the macro changes it to "term" and checks the spelling
'again. Since "term" is in the dictionary, it changes it.

'Not every word that's not in the dictionary, though, is a mistake.
'It also happens occasionally that neither form of the word
'is in the dictionary. In that case, the macro changes it back.
'So any time the macro comes across a problem letter combo
'that wasn't in the dictionary to begin with, whether it changes
'it or not, it adds a bookmark and highlights the problem word.
'This makes it easier for the user to go back and check up on the
'potential problems.

'I would eventually like to add an index that comes up in a task pane
'or a dialog box, allowing users to navigate that way rather than
'through the main body of the doc. This is the "no index" version,
'which is why all the lines that would build a table of references
'are commented out.

            .Replacement.Text = "it"
            .Forward = True
            .MatchWildcards = True
            .Wrap = wdFindStop

        End With

        Do
            'i take it iNoIndexo the loop so we can bookmark each instance as we go. This way they can be reviewed.
            
            Selection.Find.Execute
            
            If Selection.Find.Found = False Then Exit Sub

                    Selection.MoveRight Unit:=wdCharacter, Count:=1, Extend:=wdMove
                    Exit Sub
                End If
                
   
End Sub
